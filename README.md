# montage

> A living, AI-assisted knowledge graph of the EEG landscape — datasets, models, and papers, and how they connect.

## Why

- Review papers are static snapshots — a table of models goes stale the week after publication.
- The EEG field is fragmented: datasets, models, papers, and the code that implements them live in separate places with no map between them.
- "Montage" fits twice: in EEG it's the arrangement you read the brain through; in general use it's separate pieces assembled into a whole. This is that whole.

## What it captures

- A persistent, connected map of the EEG landscape — not a stream or a static table.
- Four kinds of entity: **datasets**, **models**, **papers**, **code repositories**.
- The relationships between them:
  - which models are trained and evaluated on which datasets
  - which papers produced which models
  - which papers cite which papers
  - which code repositories implement which models — official releases and community reproductions
  - which papers released which code repositories, and which repositories host which datasets
- Later: relate and cluster papers by topic and similarity.

## Purpose

- Get a genuine sense of the EEG landscape — the moving parts and how developments connect.
- Stay current as new datasets, models, and papers appear, instead of freezing at publication time.
- Useful personally as a way to navigate the field; shareable with the community.
